# デモ台本：フィールドサポートエンジニア（FSE）現場診断シナリオ

> EVS-X1000 内視鏡システム「湾曲の動きがおかしい」という曖昧な訴えから、
> 対話的診断 → 修理 → 検証 → 完了記録までを一気通貫で見せるシナリオ。
>
> **再現方法**: Codespace で `git pull && dr run dev` → ブラウザ :5173 のチャットUI（フィールド画面）。
> ユーザー入力は基本的にエージェントが提示する**クイック返信チップ（ボタン）をクリック**して進める。
> 以下の「クリック」はそのチップを押すことを指す。最初の1回だけ自由入力する。

---

## シナリオ全体像

```
症状報告（自由入力）
  → 方向確認            ：上下左右ともおかしい
  → 初手の外観・ロック・操作感確認（エージェントが提示）
  → 分岐①ロック解除でも重いか   ：一部方向だけ重い
  → 分岐②重い方向と戻り不良の一致：重い方向と戻り不良が一致した
  → 分岐③外観異常        ：亀裂や浮きがある   → CLN系へ昇格
  → 分岐④リーク圧力低下     ：圧力低下あり
  → 分岐⑤気泡箇所        ：湾曲部ゴム端部から気泡あり
  → 暫定結論（CLN-001 + INS-003 の統合診断）
  → 分岐⑥現地修理可否      ：現地で交換修理できる
  → 修理作業手順（記録→取外し→併せ点検→装着→外観確認→再リークテスト）
  → 分岐⑦再リークテスト結果   ：圧力低下なし
  → 主訴の再検証
  → 分岐⑧湾曲操作の改善     ：完全に改善した
  → 完了判断 + 作業記録
```

---

## 操作手順

### 1. 最初の症状報告（自由入力）

入力欄に以下をタイプして送信:

```
湾曲の動きがおかしい
```

→ エージェントが「どちら側の湾曲動作がおかしいですか？」と聞き、チップが出る。

### 2. 方向確認

チップ **「上下左右ともおかしい」** をクリック。

→ 全方向異常 = 単一ワイヤー断線より、ロック機構／操作部内プーリー／湾曲部ゴム／屈曲ダメージを
優先する切り分けへ。初手の確認（①外観 ②ロック解除 ③操作感 ④作業上の注意）が提示される。

### 3. 分岐①：ロック解除でも重いか

チップ **「一部方向だけ重い」** をクリック。

→ アングルワイヤーの張力不均衡／劣化途中 と 湾曲部ゴムの局所硬化・しわ寄り を優先切り分け。
INS-003 類似履歴（9件・平均63分・初回解決88.9%）が提示される。

### 4. 分岐②：重い方向と戻り不良の一致

チップ **「重い方向と戻り不良が一致した」** をクリック。

→ 重い方向系のワイヤー張力不均衡 or 湾曲部ゴム局所硬化が疑わしい。
INS-003 該当（湾曲部ゴム硬化・劣化、典型交換部品=湾曲部ゴム）。外観確認へ。

### 5. 分岐③：湾曲部ゴムの外観異常

チップ **「亀裂や浮きがある」** をクリック。

→ ★ここで CLN 系（防水・感染対策）へ昇格。
「操作部分解ではなく、漏れ箇所の特定 → 防水修理判断を優先」とリークテストへ誘導。

### 6. 分岐④：リークテスト圧力低下

チップ **「圧力低下あり」** をクリック（※チップ文言は実行時の提示に合わせる）。

→ CLN-001（リークテストで圧力低下を検出）として扱う。漏れ箇所特定テストへ。

### 7. 分岐⑤：気泡・漏れ箇所

チップ **「湾曲部ゴム端部から気泡あり」** をクリック。

→ CLN-001：湾曲部ゴム端部の防水不良 と確定的に絞り込み。
INS-003 の「重い方向と戻り不良一致」＋「しわ寄り／硬化」も合わせ、
湾曲部ゴム端部の接着・固定部劣化／端部近傍の微細亀裂が主因、湾曲抵抗は二次症状と判断。

**→ ここで「現時点の暫定結論」が提示される（主エラー・併発症状・推定原因・推奨部品・次工程）。**

### 8. 分岐⑥：現地修理可否

チップ **「現地で交換修理できる」** をクリック。

→ 修理作業手順が提示される（①交換前記録 ②水分侵入防止 ③ゴム取外し
④内部/ワイヤー系の併せ点検 ⑤新ゴム装着/端部防水 ⑥外観確認 ⑦再リークテスト）。

### 9. 分岐⑦：再リークテスト結果

チップ **「圧力低下なし」** をクリック。

→ 防水修理は一旦合格。当初の INS-003（湾曲抵抗）が交換後に解消したかの検証へ。

### 10. 分岐⑧：湾曲操作の改善

チップ **「完全に改善した」** をクリック。

→ **完了判断**（4条件をすべて満たす）と **作業記録**（発生症状・リーク結果・漏れ箇所・
修理内容・修理後確認・交換部品・追加点検）が提示され、現地修理完了として締まる。

## デモで強調したいポイント

1. **曖昧な訴えからの対話的診断**: 「湾曲の動きがおかしい」だけで開始し、エージェントが
   分岐点ごとに質問して切り分ける。FSEはチップを押すだけ。

2. **複数エラーの統合診断**: CLN-001（防水）と INS-003（湾曲抵抗）を因果関係で結びつける。
   単一ツール・単一エラーコード検索では出せない、Agentic RAG ならではの価値。

3. **CLN 系の最優先扱い**: 外観異常（亀裂・浮き）を検知した瞬間に感染対策リスク優先へ昇格。

4. **ベテラン知見の活用**: 「片側だけ戻りが遅い＝完全断線前の劣化途中」「典型交換部品は
   表記上『なし』だが現地で漏れ箇所特定後に判断するケースが多い」等、暗黙知を回答に織り込む。

5. **主訴まで遡る完了判断**: 「直して終わり」ではなく、最初の主訴（湾曲抵抗）が本当に
   解消したかまで検証ループに入る。見落としを防ぐベテランの締め方を再現。

6. **技能継承につながる作業記録**: 完了時に構造化された対応ログを生成。後任が参照できる
   記録として残せる（仕様書 7.2「対応完了をログに記録」に対応）。
